System Workflow Guide

Technical Architecture & Workflows

2all.ai is an enterprise-grade web accessibility platform built on Next.js. It features static and dynamic WCAG 2.2 evaluation engines, client verification procedures, customized user setups, and an interactive visitor interface injected directly onto customer websites.

Core Stack

Next.js 16 (App Router)
Prisma Client
SQLite / DB
Cheerio Parser
NextAuth (Beta)
Tailwind & Vanilla CSS
TypeScript
01Onboarding & Domain Verification
  • 1
    Account Registration & Login
    Users sign up or sign in to verify credentials. Password hashing is enforced via bcryptjs.
    src/app/(auth)/register/page.tsx
  • 2
    Domain Registration
    The user registers their website. The backend normalizes URLs, checks for duplicates, generates a verification token, and stores the domain state.
    src/app/api/domains/route.ts
  • 3
    Verification Execution
    The user selects a validation route (HTML upload, DNS TXT, or Meta tags). The backend fetches the live domain and verifies if the token exists, updating status to ACTIVE.
    src/app/api/domains/verify/route.ts
02Accessibility Scanner Workflow
1. Trigger API
POST /api/scan with project ID
→
2. Crawl & Fetch
Get raw website HTML response
→
3. Parse & Check
Cheerio evaluates WCAG criteria
→
4. Save & Score
Calculate score, save issues to DB
  • 1
    Crawl & Fetch Execution
    The target URL is requested in the background. A database Scan is logged as PENDING.
    src/lib/scanner/crawler.ts
  • 2
    WCAG Rule Evaluation
    HTML parser examines nodes against WCAG criteria: missing image alt tags (1.1.1), empty form input labels (1.3.1), poor contrast (1.4.3), and missing skip links (2.4.1).
    src/lib/scanner/rules.ts
  • 3
    Reporting & UI Suggestions
    Scan metrics are calculated. Identified issues are saved in DB with severities and specific suggestions, allowing users to view them instantly in their dashboard.
    src/components/dashboard/IssueList.tsx
03Widget Embedding & Loading Flow
  • 1
    Snippet Placement
    Customers inject a universal HTML script tag pointing to loader.js, supplying their data-api-key.
    src/components/dashboard/InstallCodeBlock.tsx
  • 2
    Widget Bootstrapping
    On host load, loader.js requests a validation handshake from the server. The api verifies API keys, active domain configuration, user plan limits, and logs usage stats.
    src/app/api/widget/bootstrap/route.ts
  • 3
    Asset Loading
    Once validated, loader.js dynamically injects widget-core.js and the customized settings (colors, options, layout position) onto the host website.
    public/loader.js
04End-User Accessibility Runtime adjustments

The widget loads client-side as a floating button. When clicked, it renders an Accessibility Panel built on React which uses state parameters from a global context provider to apply runtime styling and logic directly to the host webpage.

  • 1
    State & Settings Storage
    Adjustments are managed in a central state provider, immediately synced, and saved locally inside the browser's localStorage.
    src/context/AccessibilityContext.tsx
  • 2
    On-demand UI Alteration
    CSS variables are applied to scale typography sizes, increase line/letter spacing, override elements with high contrast, or apply visual filters (Protanopia / Monochromatic / Deuteranopia).
    src/components/accessibility/AccessibilityPanel.tsx
  • 3
    Advanced Assistive Logic
    Enables active utilities: Text Magnifier (zooms hovered characters), Text-to-Speech (uses speech synthesis on-click), and Read Aids (adds highlighted guide rule or mask).
    src/components/accessibility/AccessibilityWidget.tsx

Tailored Visual Profiles

Profiles (Dyslexia, Seizure Safe, Night Mode, Motor-Impaired, ADHD-friendly) bundle specific configurations together for a simplified, one-click accessibility fix that instantly adapts the DOM state to user preferences.
